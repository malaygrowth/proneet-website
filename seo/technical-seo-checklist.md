## Priority 1 — verify / fix this week

### 1. Confirm `/robots.txt` exists and allows indexing
Check `public/robots.txt`. Should contain:
```
User-agent: *
Allow: /
Disallow: /api/

Sitemap: https://proneetphysics.com/sitemap.xml
```

### 2. Verify `/sitemap.xml` renders correctly
Open `https://proneetphysics.com/sitemap.xml` in a browser. Should list all indexable pages. If Next.js isn't generating one automatically, create `app/sitemap.ts`.

### 3. Add BreadcrumbList JSON-LD
Current schema stack is missing BreadcrumbList. Add to each non-homepage route in `components/seo/json-ld.tsx`. Google renders breadcrumbs in SERPs and weights them for understanding site hierarchy.

### 4. Fix canonical URLs
Every page should declare a canonical URL via `metadata.alternates.canonical`. Currently relies on Next.js defaults. Verify and make explicit on any page with query-param variants.

### 5. Add `<link rel="alternate" hreflang="en-IN" />`
Indicates India-English locale. Helps Google serve the right SERP to Indian searchers.

### 6. Verify Core Web Vitals on production
- Run `https://pagespeed.web.dev/analysis?url=https://proneetphysics.com` after GSC verification
- Target: LCP < 2.5s, INP < 200ms, CLS < 0.1
- Hero section uses `priority` on the Neeraj portrait — good. Verify no render-blocking JS in the critical path.

### 7. Image spec audit
Every image in `public/photos/`:
- [ ] Under 100KB each (run: `find public/photos -type f -exec du -h {} \;`)
- [ ] WebP format where possible (the alumni photos are JPG — OK for photos; the scene photos are WebP)
- [ ] Descriptive alt text in every `<Image>` component

### 15. Cross-device testing
- Test every new content page on mobile (iOS Safari, Android Chrome) before publish
- Touch targets ≥ 44px
- Text readable at 16px base
- No horizontal scroll

### 16. Security headers
Verify Vercel ships:
- `Content-Security-Policy` (moderate)
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy` (restrict camera, mic, geolocation unless needed)

Add a `vercel.json` or `next.config.js` headers entry if missing.

## Watch for regressions

These are the failure modes that kill technical SEO after weeks of success:

- A dev accidentally adds `noindex` to a page (check `<meta name="robots">`)
- A dev breaks a canonical URL with a malformed query string
- A content update accidentally removes a schema block
- An image update pushes a PNG over 100KB
- A Next.js upgrade changes `sitemap` or `robots` generation defaults

Guardrail: run a pre-publish checklist (see `content-calendar.md` per-piece spec).

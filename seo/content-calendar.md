## Per-piece production spec

Each piece follows handbook §4 structure and §5 humanisation mandate. Binding checklist:

- [ ] Title tag < 60 chars, primary KW in first half
- [ ] Meta description < 155 chars with a specific number
- [ ] One H1, 70% of H2s in question format
- [ ] Definition-first opening paragraph (3-sentence pattern)
- [ ] Key Takeaways box (5–6 bullets, 3+ with numbers)
- [ ] Byline with author, role, and absolute last-updated date
- [ ] 2+ first-person experience references ("when we ran this for...")
- [ ] 1+ honest failure admission
- [ ] 1+ contrarian take with reasoning
- [ ] 1+ named expert quote (Neeraj Gupta or Vivek Patidar)
- [ ] 1+ specific client timeframe with absolute date
- [ ] 1+ named tool / method opinion with reason
- [ ] 2+ analogies from outside the industry (cricket, cooking, construction, music)
- [ ] Stats rule: every stat = number + source + year + link (or delete)
- [ ] FAQ section with 5+ Q&As, 2–3 sentence answers
- [ ] FAQPage + Article + BreadcrumbList JSON-LD
- [ ] 5+ internal links (for pillars), 3+ (satellites), absolute URLs
- [ ] Zero em dashes, zero banned words
- [ ] 1 cover image + 1 image per 800 words, WebP under 100KB, descriptive alt
- [ ] Read aloud before publish — flag any voice drift

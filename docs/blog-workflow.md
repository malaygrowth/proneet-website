# Shipping a new blog post

This is the canonical flow. Follow every step.

## 1. Write the post

Create `app/blog/<slug>/page.tsx`. Copy structure from an existing post
(e.g. `how-to-choose-a-physics-teacher`). The page should:

- Import `POST` from `BLOG_POSTS` by matching slug
- Render `<ArticleJsonLd>`, `<BreadcrumbJsonLd>`
- Use `<KeyTakeaways>`, `<PageFaq>`, `<InfographicTable>`, `<InfographicQuote>`
- Follow the Content Writer Handbook (`../CONTENT_WRITER_HANDBOOK.md`)

## 2. Register the post

Prepend an entry to `lib/blog-posts.ts` (newest-first):

```ts
{
  slug: "your-slug",
  title: "...",
  description: "...",    // 120–155 chars, used as meta description
  excerpt: "...",        // the sharpest 1–2 sentence framing of the thesis
  publishDate: "2026-MM-DD",
  lastUpdated: "2026-MM-DD",
  author: "Neeraj Gupta",
  authorRole: "Founder, ProNEET · 20+ years teaching Physics",
  category: "Pillar" | "Guide" | "Comparison" | "Location",
  readingTime: "N min read",
  targetKeyword: "the primary keyword",
  featuredImage: "/blog/your-slug/cover.webp",        // will be overwritten
  featuredImageAlt: "Cover for \"Title\".",           // will be overwritten
}
```

`featuredImage` + `featuredImageAlt` will be regenerated by the cover
pipeline — the placeholder values above are fine, just keep the shape.

## 3. Ship it

```
npm run enhance -- ship <slug>
```

This runs, in order:

1. **Cover**: GPT-4o reads the post → extracts headline, subhead, and
   3D-clay iconography → gpt-image-1 renders the illustration →
   satori composites pixel-perfect Inter typography → sharp writes
   `public/blog/<slug>/cover.webp` and rewrites the registry's
   `featuredImage` + `featuredImageAlt`.
2. **Audit**: SEO audit report to `seo/enhance-reports/<slug>-audit.md`.
3. **Section body images**: up to 6 editorial images per 120+ word
   section, written to `public/blog/<slug>/<section-slug>.webp`.
4. **Link suggestions**: internal + outbound link suggestions.
5. **Re-audit**: shows the diff in SEO score after the pipeline.

If the cover drifts (e.g. the headline picks up a sub-thesis instead
of the post's primary topic, or the iconography overlaps another
post's imagery), pin the slug in `scripts/enhance/cover-overrides.ts`
and run `npm run enhance -- covers <slug> --force`.

## 4. Verify locally

```
npm run build   # must pass
npm run dev     # visit /blog and /blog/<slug>
```

## 5. Commit and push

```
git add app/blog/<slug>/ lib/blog-posts.ts public/blog/<slug>/ seo/enhance-reports/
git commit -m "Ship <slug>: <short description>"
git push origin master
```

Vercel auto-deploys from master.

## Flags

- `--force` — regenerate cover + section images even if they already
  exist on disk. Needed when you update the post body and want fresh
  imagery.
- `--max-images N` — cap section body images per post (default 6).

## Adding more posts later

`npm run enhance -- ship <slug>` is idempotent. Running it again on a
post that already has a cover + section images is a no-op unless you
pass `--force`. Safe to re-run.
